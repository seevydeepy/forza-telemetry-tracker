**Goal:** Add GitHub Artifact Attestations to the tagged Windows desktop release path so users can verify that release assets were produced by this repository's release workflow.

**Architecture:** Keep the release artifact unsigned for Windows code-signing purposes, but add Sigstore-backed GitHub provenance attestations for the installer and checksum generated by `.github/workflows/desktop-release.yml`. Generate attestations in the Windows build job immediately after the installer and checksum are created, then verify them in the Ubuntu release job before publishing the GitHub Release. Document that attestations prove build provenance, not app safety or SmartScreen reputation.

**Tech Stack:** GitHub Actions, `actions/attest@v4`, GitHub CLI `gh attestation verify`, PowerShell release build tooling, GitHub Releases.

### Task 1: Generate Provenance Attestations In The Build Job

- [ ] **Step 1: Expand the build job token permissions**

Replace the build job permissions block:

```yaml
    permissions:

```

with:

```yaml
    permissions:
      contents: read
      id-token: write
      attestations: write

```

`id-token: write` lets the action mint the OIDC token used for Sigstore signing. `attestations: write` lets GitHub store the generated attestation. `artifact-metadata: write` matches the current `actions/attest@v4` README guidance for artifact storage metadata and is a narrowly scoped permission.

Insert this step immediately after `Create installer checksum` and before `Upload release assets`:

```yaml

        uses: actions/attest@v4
        with:
          subject-path: |
            ${{ steps.assets.outputs.installer }}
            ${{ steps.assets.outputs.checksum }}
```

This creates default SLSA provenance attestations for both release assets produced by the build job: the installer and the checksum file.

### Task 2: Verify Attestations Before Publishing The Release

**Files:**
- Modify: `.github/workflows/desktop-release.yml`

- [ ] **Step 1: Expand the release job token permissions**

Replace the release job permissions block:

```yaml
    permissions:
      contents: write
```

```yaml

      contents: write

```

`contents: write` remains necessary for `gh release create`. `attestations: read` lets the release job fetch the attestation generated by the build job for verification before publishing.

- [ ] **Step 2: Add a downloaded-asset checksum verification step**

Insert this step immediately after `Download release assets`:

```yaml
      - name: Verify downloaded release assets
        id: downloaded_assets
        shell: bash
        run: |
          set -euo pipefail
          tag_version="${GITHUB_REF_NAME#v}"
          installer="$(find release-assets -type f -name "ForzaTelemetryTrackerSetup-v${tag_version}-x64.exe" -print -quit)"
          if [[ -z "${installer}" ]]; then
            echo "Installer asset not found for ${GITHUB_REF_NAME}" >&2
            exit 1
          fi
          checksum="${installer}.sha256"
          if [[ ! -f "${checksum}" ]]; then
            echo "Checksum asset not found: ${checksum}" >&2
            exit 1
          fi
          (
            cd "$(dirname "${installer}")"
            sha256sum -c "$(basename "${checksum}")"
          )
          {
            echo "installer=${installer}"
            echo "checksum=${checksum}"
          } >> "${GITHUB_OUTPUT}"
```

This avoids assuming whether `download-artifact` restores files at `release-assets/` root or beneath a preserved directory.

- [ ] **Step 3: Add an attestation verification step**

Insert this step immediately after `Verify downloaded release assets`:

```yaml
      - name: Verify release asset attestations
        env:
          GH_TOKEN: ${{ github.token }}
        shell: bash
        run: |
          set -euo pipefail
          signer_workflow="${GITHUB_REPOSITORY}/.github/workflows/desktop-release.yml"
          source_ref="refs/tags/${GITHUB_REF_NAME}"
          gh attestation verify "${{ steps.downloaded_assets.outputs.installer }}" \
            --repo "${GITHUB_REPOSITORY}" \
            --signer-workflow "${signer_workflow}" \
            --source-ref "${source_ref}" \
            --deny-self-hosted-runners
          gh attestation verify "${{ steps.downloaded_assets.outputs.checksum }}" \
            --repo "${GITHUB_REPOSITORY}" \
            --signer-workflow "${signer_workflow}" \
            --source-ref "${source_ref}" \
            --deny-self-hosted-runners
```

This makes the release job fail before publishing if the downloaded installer or checksum is missing matching provenance from this workflow and tag.

Insert this section immediately before `## About window and updates`:

````markdown
## Artifact attestation verification

Artifact attestations prove that a downloaded asset matches a subject attested by the repository's GitHub Actions release workflow. They do not prove the application is safe, endorsed by GitHub, or signed for Windows SmartScreen.

From an empty directory, download a release and verify the installer provenance:

```powershell
gh release download v1.2.3 `
  --repo seevydeepy/forza-telemetry-tracker `
  --pattern "ForzaTelemetryTrackerSetup-v1.2.3-x64.exe*"

gh attestation verify .\ForzaTelemetryTrackerSetup-v1.2.3-x64.exe `
  --repo seevydeepy/forza-telemetry-tracker `
  --signer-workflow seevydeepy/forza-telemetry-tracker/.github/workflows/desktop-release.yml `
  --source-ref refs/tags/v1.2.3 `
  --deny-self-hosted-runners
```

Then compare the attached checksum with the downloaded installer:

```powershell
$expected = (Get-Content .\ForzaTelemetryTrackerSetup-v1.2.3-x64.exe.sha256).Split(" ")[0].ToLowerInvariant()
$actual = (Get-FileHash .\ForzaTelemetryTrackerSetup-v1.2.3-x64.exe -Algorithm SHA256).Hash.ToLowerInvariant()
if ($actual -ne $expected) {
    throw "Installer checksum mismatch: expected $expected but got $actual"
}
"Installer checksum verified: $actual"
```
````

- Spec coverage: The plan adds artifact attestations to the existing tagged release build, verifies them before publishing, documents user verification, and preserves the unsigned-installer warning.
